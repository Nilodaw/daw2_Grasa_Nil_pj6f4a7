How the application works

Architecture

  • Client (app/cli): HTML, CSS and JavaScript. Handles all user interaction and sends requests to the server.
  • Server (app/srv): PHP scripts that validate input, calculate VAT (21%), save data to the file, and return results.
  • Data file: onlineOrders/onlineOrders.db. Each line represents an order separated by |.
  • Communication: the client sends GET or POST requests; the server replies with plain text or HTML tables.

How to use

  1. Create an order: from the menu, open “Create Flower Order”, fill in all fields, and submit. The server stores the data and redirects back to the menu.
  2. List orders: choose “List All Orders” to see every stored order in a table.
  3. Search order: in “Get Order by Code”, enter a code. The JavaScript sends it to PHP, which responds with the order’s details and total (VAT included) shown in the same page.
